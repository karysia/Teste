Config user
> git config --global user.name "Paulo Proenca" 
> git config --global user.email prp@upskill.pt

Config editor
> git config --global core.editor nano

Colorize git messages
> git config --global color.ui true

Initialize repo
>git init

check the status of the repository use
> git status

start track the new file use
>git add file.js

To add all files to the index
>git add *

Commit
>git commit -m "message"

typo in your commit message, or you forget 
to stage an important change before committing:
>git commit --amend

delete file.js from repository
 git rm file.js

To move or rename use:
> git mv <source> <destination>

change the information and how it appears
> git log --pretty=format:"%h %s <%an>" --graph

only commits of a certain author
> git log --author="Paulo Proenca"

useful ways to visualize the history
> git blame file.js
> git log

stash pushes all your uncommitted changes onto a stack.
>git stahs

Compare commits
>git diff
> git diff <commit hash> (> git diff 1d2f05f..5c9d9f3a)

The .. operator signifies a range is being provided.
> git diff 1d2f05f..5c9d9f3a

To list the existing tags:
>git tag

To create a tag in the last commit:
> git tag v1.5

To create a tag in a specific commit:
> git tag v1.0 5c9d9f3
